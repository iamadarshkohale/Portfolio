AWS

EC2, S3, VPC, IAM, RDS, DynamoDB, Lambda CloudWatch, ELB, Route 53, EBS,ASG

Amazon EC2 (Elastic Compute Cloud) is a core AWS service that provides scalable virtual servers in the cloud. It allows users to run applications on virtual machines with full control over the OS, networking, and storage. You can quickly launch, stop, or scale instances based on your workload needs.

Amazon S3 (Simple Storage Service) is a scalable object storage service used to store and retrieve any amount of data from anywhere. It's ideal for backups, static website hosting, and storing media or application data. S3 offers high durability, versioning, and lifecycle policies for efficient data management.

Amazon VPC (Virtual Private Cloud) allows you to create a secure, isolated network within AWS to launch your resources. You can define subnets, route tables, and gateways to control traffic flow. It gives you full control over networking, including IP ranges, firewall rules, and internet access.

AWS IAM (Identity and Access Management) lets you securely manage access to AWS services and resources. It allows you to create users, groups, and roles, and assign fine-grained permissions using policies. IAM ensures only authorized entities can perform specific actions in your AWS environment.

Amazon RDS (Relational Database Service) is a managed service that makes it easy to set up, operate, and scale relational databases in the cloud. It supports engines like MySQL, PostgreSQL, MariaDB, Oracle, and SQL Server. RDS automates tasks like backups, patching, and replication for high availability.

Amazon DynamoDB is a fully managed NoSQL database service designed for fast and predictable performance at any scale. It supports key-value and document data models with built-in high availability and scalability. DynamoDB is ideal for real-time applications like gaming, IoT, and e-commerce.

AWS Lambda is a serverless compute service that lets you run code without provisioning or managing servers. You only pay for the compute time your code uses. Lambda automatically scales and runs your code in response to events like HTTP requests, file uploads, or database changes.

Amazon CloudWatch is a monitoring and observability service for AWS resources and applications. It collects metrics, logs, and events to help you monitor performance, detect issues, and trigger automated responses. CloudWatch enables real-time visibility and alerting to maintain application health.

AWS ELB (Elastic Load Balancing) automatically distributes incoming traffic across multiple targets like EC2 instances, containers, and IPs. It improves fault tolerance and scalability by balancing loads in one or more availability zones.

Auto Scaling Group (ASG) in AWS automatically manages the number of EC2 instances based on demand. It helps maintain performance and reduce costs by scaling out during traffic spikes and scaling in during low usage. ASG ensures high availability and fault tolerance across multiple Availability Zones.
